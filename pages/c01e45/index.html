# 用户类型

  1. root用户

    一个特殊的管理账户，也被称为超级用户

    root已接近完整的系统控制，对系统损害几乎有无限的能力

    除非必要，不然不要登录为root

  2. 普通(非特权)用户

    权限有限，并且造成损害的能力比较有限

# 终端(terminal)

设备终端：键盘、鼠标、显示器

# 终端类型

  1. 控制台终端：/dev/console

  2. 串行终端：/dev/ttyS#

  3. 虚拟终端：tty: (teletypewriters) /dev/tty# （tty可以有n个，#代表数字）

  4. 图形终端：startx,xwindows

    ​ CentOS 6: Ctrl+Alt+F7

    ​ CentOS 7: 在哪个终端启动，即位于哪个虚拟终端

  5. 伪终端：pty: (pseudo-tty) /dev/pts/# 如：ssh远程连接

# 命令提示符prompt

登录Linux后，默认的系统命令提示符无法明显辨别生产和测试环境，而导致误操作。我们可以通过修改PS1变量实现个性的提示符格式，避免这种低级错误

##CentOS默认提示符
[root@localhost ~] #
##Ubuntu默认提示符
[root@ubuntu1804 ~] $
1
2
3
4

"#"代表管理员，"$"代表普通用户

  • 显示提示符格式
[root@centos ~]$ echo $PS1
[\u@\h \W]\$
1
2
  • 修改提示符格式范例（临时生效）
PS1="\[\e[1;5;41;33m\][\u@\h \W]\\$\[\e[0m\]"
PS1="\[\e[1;32m\][\t \[\e[1;33m\]\u\[\e[35m\]@\h\[\e[1;31m\] \W\[\e[1;32m\]]\[\e[0m\]\\$"
1
2

第一条命令中"1"表示高亮，"5"表示闪烁，"41"表示背景颜色，"33"表示字体颜色，结尾的"0"表示颜色终止(否则敲出的命令和结果也会带有设定的颜色)

📕提示符格式说明

参数 描述 参数 描述
\e 控制符\033 \W 当前工作目录基名
\u 当前用户 \t 24小时时间格式
\h 主机名简称 \T 12小时时间格式
\H 主机名 ! 命令历史数
\w 当前工作目录 # 开机后命令历史数
  • 在CentOS系统实现持久保存提示符格式

​ 将PS1变量定向到/etc/profile.d文件里.sh结尾的脚本文件

[root@centos ~]$ echo 'PS1="\[\e[1;31m\][\u@\h \W]\\$\[\e[0m\]"' > /etc/profile.d/test.sh
1
  • 在Ubuntu系统持久保存提示符格式

​ 将PS1变量定向到.bashrc文件中

[peggie@ubuntu1804 ~]$ echo "PS1='\[\e[1;35m\][\u@\h \W]\\$\[\e[0m\]'" >> .bashrc
1

# 执行命令

shell中可执行的两类命令：

(1)内部命令：由shell自带的，而且通过某命令形式提供，用户登录后自动加载并常驻内存中

(2)外部命令：在文件系统路径下有对应的可执行程序文件，当执行命令时才 从磁盘加载至内存中，执行完毕后从内存中删除

  • 区别指定的命令是内部命令还是外部命令
[root@centos ~]$ type COMMAND

👇举例：
[root@centos ~]$ type hostname
hostname is hashed (/usr/bin/hostname)      #显示路径的就是外部命令
[root@centos ~]$ type echo 
echo is a shell builtin                     #显示在shell内置的就是内部命令

[root@centos ~]$ type -a cd                #查看是否存在对应内部和外部命令
cd is a shell builtin
cd is /usr/bin/cd                           
#说明cd命令既是内部命令也是外部命令（这样的命令有很多）

# Hash缓存表

系统初始hash表为空，当外部命令执行时，默认会从PATH路径下寻找该命令，找到后会将这条命令的路径记录到hash表中。当下次再使用该命令时，shell解释器会首先查看hash表，如果存在hash表内将直接执行，如果不存在hash表内将会去PATH路径下寻找。

  • hash命令常见用法
[root@centos ~]$ hash                      #显示hash缓存
[root@centos ~]$ hash -l                   #显示hash缓存，可作为输入使用
[root@centos ~]$ hash -p PATH NAME         #将命令全路径PATH起别名为NAME
[root@centos ~]$ hash -t NAME              #打印缓存中NAME的路径
[root@centos ~]$ hash -d NAME              #清除NAME缓存
[root@centos ~]$ hash -r                   #清除所有缓存
1
2
3
4
5
6
👇举例：
##第一步：查看who命令的路径
[root@centos ~]$ type who                               
who is /usr/bin/who
##第二步：执行who命令
[root@centos ~]$ who       
##第三步：再次查看who命令路径
[root@centos ~]$ type who                               
who is hashed (/usr/bin/who)       
`此时显示who命令保存在缓存中`
##第四步：将who命令从旧路径移走
[root@centos ~]$ mv /usr/bin/who /usr/local/bin      
##第五步：再次执行who命令
[root@centos ~]$ who                                    
-bash: /usr/bin/who: No such file or directory     
`此时按照旧路径寻找不到who命令`
##第六步：清除who命令的缓存
[root@centos ~]$ hash -d who            
##第七步：第三次执行who命令
[root@centos ~]$ who                                    
root     pts/0        2022-03-30 17:16 (10.0.0.1)       
`此时已经可以正常执行who命令`
##第八步：显示hash缓存
[root@centos ~]$ hash                                   
hits     command
   1     /usr/local/bin/who                             
`此时hash缓存中更新了who命令的新路径`

# 命令别名

对于经常执行的较长命令，可以将其定义成较短的别名，以方便执行

  • 显示当前shell进程所有可用的命令别名
[root@centos ~]$ alias
1
  • 定义别名NAME，其相当于执行命令VALUE
[root@centos ~]$ alias NAME='VALUE'
👇举例：
##前提是host命令当前不存在
[root@centos ~]$ alias host='hostnamectl set-hostname'     
[root@centos ~]$ host centos.peggie.org

📌举例：扫描新加的磁盘
[root@centos ~]$ alias scandisk='echo - - - > /sys/class/scsi_host/host0/scan;echo - - - > /sys/class/scsi_host/host1/scan;echo - - - > /sys/class/scsi_host/host2/scan'
##该命令用于在不关机的情况下添加新硬盘后，让系统扫描新加的硬盘（服务器中常用）
1

​ 👉注意：在命令行中定义的别名，仅对当前shell进程有效

📃总结命令执行过程：

  1. 先判断是否是别名（优先执行）

  2. 判断是否为内部命令（如果是直接执行）

  3. 判定不是内部命令则认定为外部命令

1)先到hash缓存中寻找命令（如果有直接执行）

2)到$PATH中寻找命令（如果缓存中没有）

3)command not found （如果$PATH中也没有）

# 命令格式

COMMAND [OPTIONS...] [ARGUMENTS...]
COMMAND [COMMAND] [COMMAND] ...
1
2

选项：用于启用或关闭命令的某个或某些功能

  • 短选项：UNIX风格选项，-c 例如：-l，-h
  • 长选项：GNU风格选项，--word 例如：--all，--human
  • BSD风格选项：一个字母，例如：a （使用相对较少）

参数：命令的作用对象，比如：文件名，用户名等

多个命令可以用 ";" 符号分开（非交互的批量执行）

[root@centos ~]$ who;hostname;pwd
1

注意：多个选项以及多参数和命令之间使用空白字符分隔

​ 取消和结束命令执行：==Ctrl+c（强制退出），Ctrl+d（正常退出）==

​ 一个命令可以用\分成多行

​ 选项的前后次序是有讲究的

# 会话管理

命令行的典型使用方式是，打开一个终端窗口（terminal window，以下简称"窗口"），在里面输入命令。用户与计算机的这种临时的交互，称为一次"会话"（session）。

会话的一个重要特点是，窗口与其中启动的进程是连在一起的。打开窗口，会话开始；关闭窗口，会话结束，会话内部的进程也会随之终止，不管有没有运行完

一个典型的例子就是，SSH 登录远程计算机，打开一个远程窗口执行命令。这时，网络突然断线，再次登录的时候，是找不回上一次执行的命令的。因为上一次 SSH 会话已经终止了，里面的进程也随之消失了。为了解决这个问题，会话与窗口可以"解绑"：窗口关闭时，会话并不终止，而是继续运行，等到以后需要的时候，再让会话"绑定"其他窗口

👇举例：
1.在一个窗口执行ping命令
[root@centos ~]$ ping 10.0.0.1    
2.打开另一个窗口查看ping命令的进程
[root@centos ~]$ ps aux|grep ping
root       9672  0.0  0.0 128552  1272 pts/0    S+   20:24   0:00 ping 10.0.0.1    #正在运行
root       9674  0.0  0.0 112808   960 pts/1    S+   20:24   0:00 grep --color=auto ping
3.把第一个窗口关掉后，再从第二个窗口查看ping命令的进程
[root@centos ~]$ ps aux|grep ping
root       9676  0.0  0.0 112808   964 pts/1    S+   20:26   0:00 grep --color=auto ping

# 字符集和编码

字符集与字符集编码是两个不同层面的概念

字符集(charset)：character set，二进制和字符的对应关系，不关注最终的存储形式

# nicode

Unicode是一个很大的集合，现在的规模可以容纳100多万个符号。Unicode仅仅只是一个字符集，规定了每个字符对应的二进制代码，至于这个二进制代码如何存储则没有规定

Unicode编码方案：

  • UTF-8：变长，1到4个字节
  • UTF-16：变长，2或4个字节
  • UTF-32：固定长度，4个字节

UTF-8 是目前互联网上使用最广泛的一种 Unicode 编码方式，可变长存储。使用 1 - 4 个字节表示一个字符，根据字符的不同变换长度。编码规则如下：

  • 对于单个字节的字符，第一位设为 0，后面的 7 位对应这个字符的 Unicode 码。因此，对于英文中的 0-127 号字符，与 ASCII 码完全相同。这意味着 ASCII 码的文档可用 UTF-8 编码打开。
  • 对于需要使用 N 个字节来表示的字符（N > 1），第一个字节的前 N 位都设为 1，第 N + 1 位设为0，剩余的 N - 1 个字节的前两位都设位 10，剩下的二进制位则使用这个字符的 Unicode 码来填充。

# 命令行历史

当执行命令后，系统默认会在内存记录执行过的命令。当用户正常退出时，会将内存的命令历史存放对应历史文件中，默认是~/.bash_history。登录shell时，会读取命令历史文件中记录下的命令加载到内存中，登录进shell后新执行的命令只会记录在内存的缓存区中，这些命令会在用户正常退出时“追加”至命令历史文件中。利用命令历史，可以用它来重复执行命令，提高输入效率。

  • history命令

history -anrw [filename]
history -ps arg [arg...]

-c: 清空命令历史
-d offset: 删除历史中指定的第offset个命令

-a: 追加本次会话新执行的命令历史列表至历史文件
-r: 读历史文件附加到历史列表
-w: 保存历史列表到指定的历史文件

-p: 展开历史参数成多行，但不存在历史列表中

man命令的操作方法：使用less命令实现

  • space, ^v, ^f, ^F: 向文件尾翻屏

  • b, ^b: 向文件首部翻屏

  • d, ^d: 向文件尾部翻半屏

  • u, ^u: 向文件首部翻半屏

  • RETURN, ^N, e, ^E or j or ^J: 向文件尾部翻一行

  • y or ^Y or ^P or k or ^K：向文件首部翻一行

  • q: 退出

  • #：跳转至第#行

  • 1G: 回到文件首部

  • G：翻至文件尾部

  • /KEYWORD

    以KEYWORD指定的字符串为关键字，从当前位置向文件尾部搜索；不区分字符大小写 n：下一个 N：上一个

  • ?KEYWORD 以KEYWORD指定的字符串为关键字，从当前位置向文件首部搜索；不区分字符大小写 n：跟搜索命令同方向，下一个 N：跟搜索命令反方向，上一个

# info

man常用于命令参考，GNU工具info适合通用文档参考

info没有参数，直接列出所有的页面，它的结构就像一个网站，每一页分为“节点”，链接节点之前*

📍命令格式：

info [命令]
1

导航info页

  • 方向键，PgUp，PgDn 导航
  • Tab键 移动到下一个链接
  • d 显示主题目录
  • Home 显示主题首部
  • Enter进入 选定链接
  • n/p/u/l 进入下/前/上一层/最后一个链接
  • s 文字 文本搜索
  • q 退出 info
